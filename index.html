Quick Start

Step 1

Open the project

Open this folder in VS Code:

code copilot-features-demo
Step 2

Verify Copilot

Look for the sparkle icon in the status bar. Open Chat with Ctrl+Alt+I and type “Hello”.

Step 3

Enable agent mode

In VS Code settings, set:

"chat.agent.enabled": true
Step 4

Initialize git

Create the initial snapshot:

git init && git add . && git commit -m "Initial commit"

Part A Built-in Chat Interface

Features available out of the box — no files to create, just open Chat and start typing.

Part B Customization Primitives

Create files in .github/ to shape how the AI understands your project. Version-controlled, team-shared.

Which Primitive Should I Use?

“Always follow these conventions”

Workspace Instructions — .github/copilot-instructions.md

“Python files should use type hints”

File Instructions with applyTo glob

“When writing docs, follow this style”

File Instructions with description only

“Generate tests for selected code”

Prompt File — .github/prompts/*.prompt.md

“Run this complete reporting workflow”

Skill — .github/skills/<name>/SKILL.md

“This agent can only read, not edit”

Custom Agent — .github/agents/*.agent.md

“Block edits if validation fails”

Hook — .github/hooks/*.json

“Query our Jira/database from chat”

MCP Server — .vscode/mcp.json

Appendices

A Decision Flow — Choose the right primitive 
Is this guidance that should apply to EVERY request?
├── Yes → Workspace Instructions (.github/copilot-instructions.md)
└── No
    ├── Does it apply to specific FILE TYPES?
    │   └── Yes → File Instructions with applyTo
    ├── Does it apply to specific TASKS?
    │   └── Yes → File Instructions with description only
    ├── Is it a REPEATABLE single task?
    │   └── Yes → Prompt File (.github/prompts/)
    ├── Is it a MULTI-STEP workflow with templates?
    │   └── Yes → Skill (.github/skills/)
    ├── Does it need ROLE-BASED tool restrictions?
    │   └── Yes → Custom Agent (.github/agents/)
    ├── Must behavior be DETERMINISTICALLY enforced?
    │   └── Yes → Hook (.github/hooks/)
    └── Does it need EXTERNAL tools/services?
        └── Yes → MCP Server (.vscode/mcp.json)
B Common Pitfalls & Debugging
ProblemCauseFix
Instruction not loadingVague descriptionUse "Use when..." pattern with keywords
Instruction always loadingapplyTo: "**"Use specific globs: "src/**/*.py"
Skill not appearing as /Folder name ≠ SKILL.md nameEnsure exact match
Hook not firingWrong event or JSON errorValidate JSON, check event name
YAML frontmatter failingUnescaped colons or tabsQuote descriptions: "Use when: X"
Agent not found as subagentVague descriptionAdd specific trigger phrases
MCP server unavailableConfig errorCheck .vscode/mcp.json syntax

Debugging tools: /debug (inspect prompts & context), /troubleshoot (AI log analysis), Agent Logs (View → Output → Agent channel)
