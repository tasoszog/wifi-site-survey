Scene 2 · How Copilot Works

Three Layers
of Power

Layer 1

Chat Interface

Slash commands, @participants, and #context mentions — already built in.

Layer 2

Customization

Instructions, prompts, hooks, agents, and skills — files you create and share.

Layer 3

External Tools

MCP servers and extensions — connect Copilot to external systems.

Narration Script

Copilot's power comes from three layers. First, the built-in chat interface — slash commands, at-participants, and context mentions that are already available. Second, customization primitives — instructions, prompts, hooks, agents, and skills that you create as files in your project. And third, external tools through MCP servers and extensions.

Narration Script

To keep things practical, we use a simple WiFi site-survey toolkit as our domain. It includes sample access point data, Python utilities, validation scripts, and a data specification. The domain is intentionally simple — the focus is entirely on the Copilot features, not the application code.
